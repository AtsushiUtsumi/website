jsonはオブジェクトを何でもブレースで囲めるので最初の要素にtypeを必ず持つようにして
jsonのもともと持っている string, number, boliean は「jsonObjectType: "string"」,「jsonObjectType: "number"」,「jsonObjectType: "boliean"」

		{
			jsonObjectType: "function",
			name: "hoge",
			params: ""
			lines: [{},{},{},{}],
		}

クラスのコードジェネレータを作成中に思った。変数のデフォルト値を設定するのはメソッドの実装と同じ

基準はブレース({})1つのjsonファイルから1つの対象言語のソースコードファイル(なんならjson自身をコピーして文字列置換で作成してもおもしろい) 

	{[
		{template: switch},
		{template: for}
	]}

↑これは実装の話。ネストがあるので面倒くさい。ひょっとしてテンプレートエンジンってネストできないんじゃね?

↓これは抽象の話。


{[
	{template: class, template_params:{}},
	{template: function, template_params:{}},
]}

template=classなので「java_class.template」に以降のパラメータ(template_params)を渡す
ジェネレータはプログラマが手を付けないファイルを自動生成しないといけないので(でないと上書きされる!)
